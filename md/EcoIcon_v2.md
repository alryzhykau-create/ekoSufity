# EcoIcon — система иконок ekoSufity

## 1. Назначение

**EcoIcon** — это единая система работы с иконками для сайта ekoSufity.

Цель EcoIcon: чтобы все иконки сайта выглядели как одна бренд-система, а не как случайные картинки из разных генераций.

Внутри EcoIcon есть два режима:

1. **EcoIcon Extract** — аккуратная техническая нарезка уже утверждённых иконок из превью без перерисовки.
2. **EcoIcon Create** — создание новых иконок в том же утверждённом premium gold стиле, когда нужной иконки ещё нет.

Главное правило: **не смешивать нарезку и генерацию**.

Если иконка уже утверждена — её нужно только нарезать.  
Если нужной иконки ещё нет — её можно создать, но строго по стандарту EcoIcon.

---

Этот блок можно вставить прямо в инструкции проекта ChatGPT / ekoSufity:

```text
Для всех задач по иконкам использовать систему EcoIcon.

EcoIcon Extract — если иконка уже утверждена и её нужно нарезать из превью. Не перерисовывать, не менять стиль, не добавлять фон, круги, рамки, glow и новые детали.

EcoIcon Create — если нужна новая иконка. Создавать только в утверждённом стиле ekoSufity: premium duotone gold, #C98A3A, мягкие золотые градиенты, transparent background, без текста, без квадратной подложки, без грязного glow, 512×512, одинаковая визуальная масса.

Не смешивать нарезку и генерацию. Если есть утверждённый референс — сначала использовать Extract.

Если пользователь просит не SVG, а “картинки”, “мини-сцены”, “AVIF для сайта”, использовать EcoIcon MiniScene: 512×512, AVIF/WebP/PNG, мягкий beige-gold стиль, без текста, без встроенного скругления, без рамки, один крупный объект, минимум деталей, CSS делает border-radius и shadow.
```

# Часть A. EcoIcon Extract

## 3. Когда использовать EcoIcon Extract

Использовать **EcoIcon Extract**, если пользователь говорит:

- “нарежь”;
- “вырежи отдельно”;
- “из утверждённого превью”;
- “не меняй стиль”;
- “как на референсе”;
- “дай отдельные файлы”;
- “сделай ZIP из этих иконок”;
- “убери фон, круг, текст, соседние элементы”.

EcoIcon Extract — это не генерация.  
Это техническое извлечение уже утверждённых иконок.

---

## 4. Главное правило Extract

Если есть утверждённое превью, нельзя заново генерировать иконки через image generation.

Нужно брать утверждённое превью и извлекать из него каждую иконку как отдельный файл.

Запрещено:

- перерисовывать иконку;
- менять стиль;
- менять цвет;
- добавлять квадратный фон;
- добавлять круг, если он не является частью самой утверждённой иконки;
- добавлять glow, тени, рамки, обводки;
- улучшать дизайн “на глаз”;
- придумывать недостающие детали;
- растягивать иконку;
- обрезать края;
- оставлять соседние элементы;
- оставлять текстовые подписи;
- использовать случайные AI-generated версии вместо утверждённого референса.

Разрешено:

- аккуратно вырезать символ;
- убрать внешний фон;
- очистить края;
- центрировать на квадратном холсте;
- добавить безопасный прозрачный отступ;
- экспортировать в PNG/WebP;
- собрать ZIP;
- собрать контрольный preview из уже экспортированных файлов.

---

## 5. Какие входные данные нужны для Extract

Для качественной нарезки лучше всего получить:

1. Утверждённое превью иконок в максимальном разрешении.
2. Желательно PNG, WebP или PDF без сильного сжатия.
3. Лучше один файл, где все иконки видны крупно и чётко.
4. Если иконок много — желательно пронумерованный скриншот или список названий по порядку.
5. Чёткое указание, что именно нужно вырезать:
   - только пиктограмму;
   - пиктограмму с внутренней мягкой подложкой;
   - без круга;
   - без текста;
   - без соседних элементов.

Если пользователь присылает маленький скриншот, сильно сжатую картинку или картинку с иконками вплотную друг к другу, результат может быть хуже. В таком случае нужно честно сказать, что нужен файл крупнее или исходник.

## 6. Правильная команда для Extract

```text
Используй EcoIcon Extract.

Это задача не на генерацию, а на аккуратную нарезку утверждённых иконок.

Возьми утверждённое превью, вырежи каждую иконку отдельно, не меняй стиль, не перерисовывай, не добавляй фон, круги, рамки, glow и новые детали.

Формат:
- PNG/WebP
- 512×512
- прозрачный фон
- иконка по центру
- одинаковая визуальная масса
- без текста
- без соседних элементов

Собери ZIP и контрольный preview только из экспортированных файлов.
```

## 7. Алгоритм Extract

### 7.1. Определить, что задача именно на нарезку

Перед началом проверить: пользователь хочет извлечь утверждённые иконки, а не создать новые.

Если пользователь говорит “нарежь”, “вырежи”, “из утверждённого превью”, “не меняй стиль” — использовать extraction workflow, а не генерацию.

---

### 7.2. Найти границы каждой иконки

Нужно определять границы не по квадратной карточке, а по реальному визуальному объекту.

Что считается объектом:

- золотая пиктограмма;
- мягкая внутренняя подложка, если она является частью утверждённого стиля;
- аккуратный glow только если он уже есть в утверждённом превью и явно является частью дизайна.

Что не считается объектом:

- белый/серый фон превью;
- сетка;
- текстовая подпись;
- соседняя иконка;
- внешний CSS-круг, если он не должен быть частью файла;
- прямоугольная карточка;
- скриншотный фон страницы.

---

### 7.3. Вырезать с запасом

Нельзя резать вплотную по краю символа.

Нужно оставить прозрачный safe padding:

- минимум 10–14% от размера объекта;
- для 512×512 итогового холста визуальный объект обычно должен занимать 330–390 px по большей стороне;
- если иконка тонкая или вытянутая, нужно выравнивать её по оптической массе, а не по математическому размеру.

---

### 7.4. Удалить фон

Финальный файл должен иметь прозрачный внешний фон.

Если исходник имеет однотонный фон:

- удалить фон через цветовой ключ или маску;
- проверить края вручную;
- не оставлять белую/серую кайму;
- не оставлять квадратную область вокруг иконки.

Если фон сложный или похож на цвета иконки:

- не делать агрессивное автоматическое удаление;
- лучше попросить более чистый исходник или нарезать вручную по областям.

---

### 7.5. Центрировать на квадратном холсте

Каждая иконка должна быть помещена на прозрачный квадратный холст.

Стандарт:

- размер холста: 512×512 px;
- фон: transparent;
- объект: по центру;
- визуальная масса: одинаковая во всём наборе;
- без подписи;
- без соседних элементов.

Центрирование должно быть не только математическим, но и визуальным. Например, если у иконки тяжёлая нижняя часть, её можно чуть поднять, чтобы она казалась центрированной.

---

### 7.6. Экспортировать файлы

Основной экспорт:

- PNG 512×512 transparent;
- WebP 512×512 transparent;
- ZIP со всеми файлами;
- контрольный preview из экспортированных файлов.

Имена файлов должны быть чистыми и понятными.

Пример:

```text
hero-pomiar-doradztwo.png
hero-wroclaw-okolice.png
hero-materialy-premium.png
hero-kontakt-bez-bariery.png
rozwiazania-sufit-napinany.png
rozwiazania-oswietlenie.png
rozwiazania-linie-led.png
rozwiazania-ukryty-karnisz.png
proces-kontakt.png
proces-pomiar.png
proces-wycena.png
proces-montaz.png
social-facebook.png
social-instagram.png
social-tiktok.png
social-youtube.png
```

---

## 8. Контрольный preview

После экспорта нужно собрать отдельную проверочную картинку, где показаны только уже нарезанные файлы.

Preview нужен, чтобы пользователь увидел:

- нет ли фона;
- нет ли квадратов;
- одинаковый ли размер;
- не обрезаны ли края;
- не попали ли соседние элементы;
- не отличаются ли цвета;
- нормально ли иконки смотрятся рядом.

Важно: preview должен быть собран только из финальных экспортированных файлов, а не из исходника.

---

## 9. Чеклист качества Extract

Перед выдачей проверить каждую иконку:

```text
[ ] Файл 512×512 px.
[ ] Внешний фон прозрачный.
[ ] Нет белого/серого квадрата.
[ ] Нет подписи.
[ ] Нет соседних элементов.
[ ] Иконка не обрезана.
[ ] Иконка не растянута.
[ ] Иконка визуально по центру.
[ ] Размер похож на остальные иконки.
[ ] Цвет не изменён.
[ ] Стиль не изменён.
[ ] Glow/подложка сохранены только если они были частью утверждённого референса.
[ ] Preview собран из экспортированных файлов.
```

Если хотя бы один пункт не проходит — исправить до выдачи.

---

## 10. Что делать, если исходник плохой

Если в исходнике:

- иконки маленькие;
- сильная компрессия;
- есть мыло;
- фон смешан с иконкой;
- иконки перекрываются;
- нет чётких границ;
- часть иконки обрезана;

нужно не фантазировать, а честно сказать:

```text
Из этого исходника можно сделать только приблизительную нарезку. Для чистого результата нужен исходник крупнее или отдельный утверждённый preview без сжатия.
```

Можно сделать черновую нарезку, но нельзя утверждать, что она идеальная.

# Часть B. EcoIcon Create

## 11. Когда использовать EcoIcon Create

Использовать **EcoIcon Create**, если пользователь говорит:

- “создай новую иконку”;
- “нужна иконка для нового блока”;
- “сделай в этом же стиле”;
- “добавь иконку для новой услуги”;
- “иконки ещё нет в утверждённом наборе”.

EcoIcon Create используется только когда нужной иконки ещё нет.

---

## 12. Стиль EcoIcon Create

Правила создания:

- стиль: premium duotone gold;
- основной цвет: `#C98A3A`;
- мягкие золотые градиенты;
- прозрачный внешний фон;
- без квадратной подложки внутри файла;
- без текста внутри иконки;
- без грязного glow;
- без случайных декоративных деталей;
- одинаковая оптическая масса с уже утверждёнными иконками;
- простая понятная метафора;
- не больше 1–2 ключевых объектов внутри одной иконки;
- финальный экспорт PNG/WebP 512×512 transparent;
- желательно SVG-версия для сайта, если иконка должна быть масштабируемой.

---

## 13. Цветовой стандарт

Основной фирменный gold:

```text
#C98A3A
```

Допустимые дополнительные оттенки для мягкого premium duotone:

```text
Light gold: #F0C879
Dark gold:  #8A5524
Soft gold:  #D9A85C
```

Запрещено:

- кислотно-жёлтый;
- грязный оранжевый;
- слишком яркий glow;
- случайные разноцветные детали;
- чёрные или серые детали внутри иконки без необходимости;
- фоновые квадраты и плашки, если пользователь не просил.

---

## 14. Геометрия и визуальный стандарт

Каждая новая иконка должна быть:

- простой;
- читаемой на маленьком размере;
- центрированной;
- с мягкой округлой геометрией;
- без перегруженных деталей;
- визуально равной остальным иконкам;
- без текста;
- без мелких элементов, которые исчезают на мобильном;
- без реалистичной 3D-сцены.

Формат:

```text
Canvas: 512×512 px
Background: transparent
Object area: примерно 330–390 px по большей стороне
Safe padding: 10–14%
Optical mass: одинаковая с остальными
```

## 16. Алгоритм Create

### 16.1. Определить смысл

Перед созданием новой иконки нужно понять:

1. Для какого блока сайта нужна иконка.
2. Какой смысл она должна передавать.
3. Есть ли похожая утверждённая иконка, чтобы повторить стиль.
4. Нужно ли делать только визуализацию или сразу финальные файлы.

---

### 16.2. Выбрать простую метафору

Иконка должна передавать один главный смысл.

Примеры:

```text
Pomiar i doradztwo — miarka + mały znacznik
Wrocław i okolice — pin/mapa + okrąg 100 km
Materiały premium — warstwa materiału + gwiazdka jakości
Kontakt bez bariery — dymki rozmowy + języki
Sufit napinany — prosty kontur pokoju z napiętym sufitem
Oświetlenie — punktowe światła
Linie LED — prosta linia świetlna
Ukryty karnisz — zasłona + ukryta szyna
Montaż bez remontu — narzędzie + czysta powierzchnia
Gwarancja — tarcza + znak jakości
```

---

### 16.3. Не перегружать

Плохая иконка:

```text
слишком много предметов + текст + фон + блики + тени + реализм
```

Хорошая иконка:

```text
1–2 понятных объекта + чистая форма + единый gold стиль + прозрачный фон
```

# Часть C. Использование на сайте

## 18. Как подключать иконки в коде

Для сайта лучше использовать одинаковые контейнеры.

Если иконки в PNG/WebP:

```css
.eko-icon-box {
  width: 72px;
  height: 72px;
  display: flex;
  align-items: center;
  justify-content: center;
}

.eko-icon-box img {
  width: 56px;
  height: 56px;
  object-fit: contain;
  display: block;
}
```

Если нужна подложка, её делает CSS-контейнер, а не сама иконка:

```css
.eko-icon-card {
  width: 76px;
  height: 76px;
  border-radius: 22px;
  background: linear-gradient(
    145deg,
    rgba(201, 138, 58, 0.12),
    rgba(240, 200, 121, 0.06)
  );
  display: flex;
  align-items: center;
  justify-content: center;
}
```

Сама иконка при этом остаётся прозрачной.

---

## 19. Что нельзя делать в коде сайта

Запрещено:

- вставлять иконки через `object-fit: cover`;
- обрезать иконки контейнером;
- добавлять фон внутрь PNG;
- дублировать CSS-круг внутри файла;
- менять цвет фильтрами без проверки;
- использовать разные размеры контейнеров;
- растягивать иконки по ширине или высоте;
- добавлять случайные тени поверх всех иконок;
- использовать PNG/WebP, если лучше подходит чистый SVG.

Правильно:

- `object-fit: contain`;
- одинаковые контейнеры;
- прозрачный файл;
- фон/карточка/круг только через CSS;
- одинаковая визуальная масса;
- одинаковые отступы.

# Часть D. Рабочий процесс

## 20. Минимальный workflow Extract

1. Открыть исходное preview.
2. Определить области иконок.
3. Для каждой области сделать crop.
4. Удалить внешний фон.
5. Очистить края.
6. Масштабировать объект на прозрачный холст 512×512.
7. Центрировать по оптической массе.
8. Экспортировать PNG/WebP.
9. Собрать ZIP.
10. Собрать preview из экспортированных файлов.
11. Проверить чеклист.
12. Отдать файлы.

---

## 21. Минимальный workflow Create

1. Определить смысл новой иконки.
2. Найти ближайший утверждённый референс по стилю.
3. Выбрать простую метафору.
4. Создать иконку в premium duotone gold стиле.
5. Убрать текст, квадрат, лишний glow и случайные детали.
6. Поместить на прозрачный холст 512×512.
7. Сравнить с существующими иконками по визуальной массе.
8. Собрать preview рядом с другими иконками.
9. Проверить чеклист.
10. Отдать PNG/WebP, при возможности SVG.

# Часть E. EcoIcon MiniScene — мягкие картинки для сайта

## 25. Когда использовать EcoIcon MiniScene

**EcoIcon MiniScene** использовать, когда иконка на сайте должна выглядеть не как SVG/line-icon, а как маленькая премиальная картинка или мини-сцена.

Этот режим подходит для блоков, где визуал должен быть мягким, дорогим и понятным, но не должен спорить с текстом.

Использовать MiniScene, если пользователь говорит:

- “продолжим как картинки”;
- “не SVG”;
- “картинки выглядят дороже”;
- “мини-сцены”;
- “для сайта в формате AVIF”;
- “512×512”;
- “без карточки / без встроенного фона / скругление через CSS”.

MiniScene — это не обычная иконка на прозрачном фоне. Это маленькая визуальная сцена с одним главным объектом и мягкой атмосферой.

---

## 26. Главное правило MiniScene

Нельзя генерировать большой общий board и потом нарезать его на отдельные картинки.

Правильно:

```text
1 генерация = 1 финальная картинка
1 файл = 1 смысл
1 картинка = 1 главный объект или одна простая сцена
```

Запрещено:

- резать отдельные картинки из общего board;
- оставлять подписи внутри изображения;
- оставлять соседние элементы;
- делать встроенные скругления углов;
- делать встроенные рамки;
- делать встроенную карточку, если пользователь просит “без фона”;
- делать слишком много мелких деталей;
- использовать жёсткий контраст;
- использовать грязный glow;
- смешивать фото, 3D, line-icon и SVG в одном наборе;
- менять уже утверждённый стиль от картинки к картинке.

Разрешено:

- мягкий нейтральный studio/background glow вокруг объекта, если это часть визуала;
- лёгкая тень под объектом;
- 1–2 крупных смысловых объекта;
- мягкий premium beige/gold стиль;
- AVIF/WebP/PNG экспорт;
- CSS-скругление и CSS-тень на сайте.

---

## 27. Стиль MiniScene

Стиль:

```text
premium soft beige-gold mini-scene
warm beige and cream palette
soft gold accents
low contrast
clean studio lighting
minimal details
one large readable object
no text inside image
no rounded corners inside file
no frame inside file
no card background inside file
CSS controls radius, shadow and crop
```

Цветовое направление:

```text
Основной gold: #C98A3A
Soft gold: #D9A55C
Cream: #FFF7EC
Warm beige: #E8D2B3
Dark accent only if needed: #2A241E
```

Важно: MiniScene должна выглядеть дороже не за счёт количества деталей, а за счёт света, композиции, чистоты и единого тона.

---

## 28. Формат MiniScene для сайта

Основной формат для маленьких картинок сайта:

```text
Canvas: 512×512 px
Format: AVIF для сайта
Fallback: WebP
Reserve/source: PNG
Aspect ratio: 1:1
Text inside image: no
Rounded corners: no
Border: no
Embedded card: no
CSS handles border-radius, shadow and background
```

Рекомендуемая структура:

```text
/images/home/[block]/avif/name.avif
/images/home/[block]/webp/name.webp
/images/home/[block]/png/name.png
```

Пример:

```text
/images/home/hero-usp/avif/hero-bezplatny-pomiar.avif
/images/home/hero-usp/webp/hero-bezplatny-pomiar.webp
/images/home/hero-usp/png/hero-bezplatny-pomiar.png
```

---

## 29. CSS для MiniScene на сайте

Сама картинка должна быть прямоугольной/квадратной без встроенного скругления. Скругление делает сайт.

```css
.ecoMiniScene {
  width: 100%;
  aspect-ratio: 1 / 1;
  object-fit: cover;
  border-radius: 14px;
  display: block;
}
```

Если картинка используется как маленький элемент рядом с текстом:

```css
.ecoMiniSceneSmall {
  width: 48px;
  height: 48px;
  object-fit: cover;
  border-radius: 12px;
  display: block;
}
```

## 30. Hero USP MiniScene — утверждённый подход

Для блока Hero USP на главной странице ekoSufity использовать мягкие маленькие картинки, если SVG/line-icons выглядят слишком сухо или дёшево.

Пункты:

```text
hero-bezplatny-pomiar
hero-dojazd-100km
hero-15-lat-gwarancji
hero-czysty-montaz
```

Правила для Hero USP:

```text
512×512
AVIF + WebP + PNG
без текста внутри картинки
без скругления
без рамки
без встроенной карточки
один крупный смысловой объект
минимум деталей
мягкий светлый beige-gold стиль
низкий контраст
скругление делает CSS
```

Смысловые образы:

| Файл | Образ |
|---|---|
| `hero-bezplatny-pomiar` | лазерный дальномер + план/комната + один луч |
| `hero-dojazd-100km` | фургон/дорога + крупный pin/маршрут, без сложной карты |
| `hero-15-lat-gwarancji` | щит/сертификат + 15 LAT как главный смысл |
| `hero-czysty-montaz` | пылесос/чистая комната/аккуратный монтаж без мусора |

Важно: для маленьких картинок не добавлять много мелких деталей. Детали не читаются и превращаются в шум. Дорого выглядит крупный объект, мягкий свет и чистая композиция.

## 31. Правильная команда для MiniScene

```text
Используй EcoIcon MiniScene.

Создай одну отдельную картинку для [название блока / смысл].

Формат:
- 512×512 px
- 1:1
- без текста внутри изображения
- без скруглений
- без рамки
- без встроенной карточки
- AVIF/WebP/PNG на финале

Стиль:
- premium soft beige-gold mini-scene
- мягкий светлый фон/атмосфера
- низкий контраст
- один крупный главный объект
- минимум мелких деталей
- #C98A3A как золотой акцент
- CSS потом делает border-radius и shadow

Не делай общий board. Не нарезай из board. Одна генерация = одна финальная картинка.
```

## 32. Чеклист качества MiniScene

```text
[ ] Файл 512×512 px.
[ ] Нет текста внутри картинки.
[ ] Нет встроенного border-radius.
[ ] Нет рамки.
[ ] Нет карточки/плашки, если пользователь просил “без фона”.
[ ] Один главный смысловой объект читается сразу.
[ ] Нет лишних мелких деталей.
[ ] Цвета мягкие, без жёсткого контраста.
[ ] Стиль совпадает с остальными MiniScene.
[ ] Картинка не выглядит как случайное фото.
[ ] Картинка не выглядит как сухой SVG.
[ ] Объект не обрезан.
[ ] Есть safe padding.
[ ] Экспортированы AVIF + WebP + PNG.
[ ] Preview собран из финальных файлов.
```

Если MiniScene используется рядом с текстом, картинка должна быть второстепенной и не спорить с текстом.

---

## 33. Когда НЕ использовать MiniScene

Не использовать MiniScene, если:

- нужна чистая пиктограмма на прозрачном фоне;
- иконка должна быть SVG line-icon;
- нужен логотип соцсети;
- пользователь прямо просит transparent icon без фона;
- иконка стоит в очень маленьком размере 16–24 px;
- нужен технический контур без объёма.

В этих случаях использовать EcoIcon Create / Extract / SVG по ситуации.

---

## 34. Новый выбор режима

Перед началом всегда определить режим:

```text
EcoIcon Extract — нарезать утверждённый референс без изменений.
EcoIcon Create — создать новую premium gold иконку на прозрачном фоне.
EcoIcon MiniScene — создать мягкую 512×512 картинку/мини-сцену для сайта.
EcoIcon QA — проверить готовый набор.
```

Если пользователь недоволен SVG и говорит “давай картинки”, переключаться на **EcoIcon MiniScene**, а не пытаться снова улучшать SVG.



## 35. Главное резюме

EcoIcon — это не “попросить нейросеть нарисовать иконку”.

EcoIcon — это система:

```text
Extract: утверждённая иконка → аккуратная нарезка → transparent 512×512
Create: новая иконка → строго тот же premium gold стиль → transparent 512×512
MiniScene: мягкая картинка для сайта → 512×512 AVIF/WebP/PNG без текста и встроенного скругления
QA: проверка результата → нет фона, квадратов, кривых краёв и разного масштаба
```

Если иконка уже утверждена — не генерировать.  
Если иконки ещё нет — создать в стиле EcoIcon.  
Если результат плохой — проверить по чеклисту, а не “улучшать на глаз”.
