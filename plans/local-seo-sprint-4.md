The implementation must preserve existing routes and slugs. No new dependencies, route changes, or redesigns are allowed.

## Context and Orientation

This project uses Next.js App Router. The listing page for local SEO is `app/lokalizacje/page.tsx`. Individual local pages are generated by `app/lokalizacje/[slug]/page.tsx`, which reads `locations` from `lib/locations.ts`, uses `generateStaticParams` to create static pages, and uses `generateMetadata` with `createPageMetadata` for canonical metadata.

The detail layout is `components/LocationPageTemplate.tsx`. It renders a JSON-LD `Service`, breadcrumbs, `LocationHero`, local service sections, pricing links, process, gallery, FAQ, CTA, internal links, and nearby locations. FAQ schema is generated through `LocalFAQSection`, which delegates to `FAQSection` and `FAQJsonLd`.

## Plan of Work

First, extend the `LocationItem` type with a `localDescription` field and add two to four unique paragraphs for each existing location. These paragraphs must follow the user-provided local emphasis for each district or city and avoid copy-paste city-name swaps.

Second, update `components/LocationPageTemplate.tsx` to render the new local paragraphs in the local-description area while preserving the existing visual rhythm. Also change the gallery heading and description to neutral wording such as “Przykładowe inspiracje dla podobnych wnętrz” when there is no confirmed local gallery content.

Third, run a small programmatic QA script against the constants and generated output. It should confirm all expected slugs exist, metadata are non-empty and unique, FAQ arrays have at least five local questions, nearby links do not include the current page, and no forbidden placeholder text is present in local constants.

Finally, run `npm run lint` and `npm run build`. Report known `<img>` warnings if they remain non-blocking.

## Validation and Acceptance

Acceptance requires `npm run build` to generate all local pages successfully. Each local page must have one H1, unique metadata, visible local copy, visible price links to `/ceny` and `/ceny#calculator`, FAQ JSON-LD via `FAQSection`, breadcrumbs, and nearby locations excluding the current page.
