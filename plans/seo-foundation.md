## Context and Orientation

This is a Next.js 14 App Router project using TypeScript and Tailwind CSS. The main landing page lives in `app/page.tsx` and is composed from section components under `components/sections`. Root metadata lives in `app/layout.tsx`. FAQ content currently lives in `components/sections/FAQSection.tsx`. Floating and section CTAs point to phone, WhatsApp, pricing, calculator, gallery, and contact anchors.

JSON-LD is a structured-data format embedded in a script tag with `type="application/ld+json"`. It does not change what visitors see, but helps search engines understand entities such as a local business, a breadcrumb trail, and frequently asked questions. A sitemap is an XML URL list generated by Next.js from `app/sitemap.ts`. A robots file tells crawlers which URLs are allowed and where the sitemap lives.

## Plan of Work

First, add shared site constants in a small module so metadata, sitemap, robots, and JSON-LD all use the same canonical URL, business name, phone number, email, and address. Export FAQ items from `components/sections/FAQSection.tsx` so structured data can reuse the exact customer-facing Polish questions and answers already rendered on the page.

Next, expand `app/layout.tsx` metadata with canonical URL, richer title and description, Open Graph, robots directives, and Polish locale information. Add `app/sitemap.ts` and `app/robots.ts` using the same canonical URL.

Then add a small server component for JSON-LD scripts and include it in `app/page.tsx`. The JSON-LD should cover `LocalBusiness`, `Service`, `FAQPage`, and `BreadcrumbList`. The visible page should remain visually stable; if a breadcrumb element is added, keep it screen-reader oriented or very restrained so it does not compete with the hero.

Finally, strengthen the footer with clear CTAs such as WhatsApp and free measurement while preserving the current light premium style and existing anchors.
